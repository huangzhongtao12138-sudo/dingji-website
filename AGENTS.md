# 鼎基装饰官网维护规则

本仓库是“鼎基装饰官网”的长期维护项目。所有后续修改必须基于现有文件、页面结构、品牌风格和历史决策继续进行，不因缺少旧聊天记录而推翻已有成果。

## 每次任务开始前

1. 完整检查项目目录和与任务相关的页面、样式、脚本及资源。
2. 读取 `docs/PROJECT_MEMORY.md`，了解公司资料、网站结构、技术与部署方式。
3. 读取 `docs/DECISIONS.md`，避免重复讨论或推翻已确认决策。
4. 检查 `git status` 和近期提交，保留用户尚未提交的改动。
5. 明确本次涉及的页面和文件后再编辑。

## 内容与事实

- 公司名称、电话、邮箱、地址、成立年份、业务范围和资质等信息，以现有官网和 `docs/PROJECT_MEMORY.md` 中标记为“已确认”的资料为准。
- 不虚构奖项、资质、客户、项目数量、营收、规模、工期承诺或其他经营数据。
- 未确认的事实必须明确标记为待确认，或先向用户询问。
- 文案使用专业、可信、自然的中文，避免无法验证的夸张表达。
- 公司名称、品牌名、案例名称与联系方式必须保持一致。

## 设计与交互

- 延续现有简洁、高级、专业的装饰工程企业视觉语言。
- 核心品牌红为 `#f30700`；继续使用纸张米灰背景、玻璃质感卡片、粗体中文标题和现有圆角体系。
- 优先复用现有页面结构、组件样式和交互，不擅自重写整个页面。
- 图片主体必须清晰、裁切合理；同时检查桌面端和手机端。
- 手机端不得出现非预期横向溢出，导航和主要操作必须可用。

## 代码、SEO 与部署

- 这是纯静态网站，生产文件全部位于 `outputs/`。
- 生产首页是 `outputs/index.html`。
- Vercel Output Directory 是 `outputs`；推送 `main` 会触发正式网站自动部署。
- 正式主域名是 `https://www.djzhuangshi.com/`，canonical 和 sitemap 使用 `www`。
- 新增或删除可收录页面时，同步更新：
  - 页面 `<title>`、description、canonical 和必要的 Open Graph 信息；
  - 合适的 JSON-LD；
  - 站内导航与相关页面链接；
  - `outputs/sitemap.xml`；
  - `docs/PROJECT_MEMORY.md` 的页面清单。
- 重要页面标题和链接文字应简洁、明确，便于搜索引擎理解站点结构。
- 不承诺 Google 一定展示站点子链接；它由搜索引擎自动决定。

## 联系信息修改清单

修改公司电话、邮箱或地址时，至少检查：

- `outputs/index.html`
- `outputs/dingji-contact.html`
- `outputs/dingji-service.html`
- `outputs/dingji-consulting.html`
- 首页与联系页中的 JSON-LD
- `docs/PROJECT_MEMORY.md`

## 完成任务前

1. 检查内部链接、资源路径、HTML 标题与 description。
2. 验证所有 JSON-LD 可以解析，验证 `outputs/sitemap.xml`。
3. 检查桌面端与 390px 左右手机端布局。
4. 运行 `git diff --check`，确认没有误改或遗漏文件。
5. 清楚说明哪些内容已完成、是否已经提交、推送和上线。
6. 生产部署属于实际线上变更；推送前应明确告知用户会触发 Vercel 自动部署。

## 长期记忆维护

- 新的已确认公司资料、页面结构或维护规则：更新 `docs/PROJECT_MEMORY.md`。
- 对未来有影响的重要选择：追加到 `docs/DECISIONS.md`。
- 记忆文件只保存稳定、可复用、已确认的信息，不保存临时聊天内容或敏感信息。
